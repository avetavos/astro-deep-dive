## TypeScript มีมาให้ในตัว

Astro มาพร้อม TypeScript support ในตัว — เขียนไฟล์ `.ts`, type component script และได้ type-checking ใน editor โดยไม่ต้องตั้งอะไร โปรเจกต์ใหม่ extend preset ตัวหนึ่งของ Astro ใน `tsconfig.json`:

```json
{
  "extends": "astro/tsconfigs/strict"
}
```

preset มี `base`, `strict` และ `strictest` — เลือก `strict` (หรือ `strictest`) สำหรับโปรเจกต์จริง มันเปิด check ที่จับ bug จริง ๆ

## การ type props ของ component

คุณ type props ของ component ด้วยการประกาศ interface `Props` ใน frontmatter Astro ใช้มัน type-check ทุกที่ที่ component ถูกใช้:

```astro
---
interface Props {
  title: string;
  count?: number;      // optional prop
}
const { title, count = 0 } = Astro.props;
---
<h1>{title} ({count})</h1>
```

ส่ง type ผิดหรือลืม prop ที่จำเป็น editor (และ `astro check`) จะเตือนที่จุดเรียกใช้

## type ที่ generate จาก content schema

ตรงนี้แหละที่ typing ของ Astro โดดเด่น เมื่อคุณ define content collection ด้วย Zod schema (จากโมดูล content) Astro **generate TypeScript type** จาก schema นั้น `getCollection` และ `getEntry` จะคืน entry ที่มี type ครบ — editor รู้ทุก field ใน frontmatter และ type ของมัน:

```ts
import { getCollection } from 'astro:content';

const posts = await getCollection('blog');
// posts[0].data.title  -> string    (from the schema)
// posts[0].data.pubDate -> Date     (z.coerce.date())
// posts[0].data.nope    -> type error: property does not exist
```

## การ type Astro.locals

เมื่อใช้ middleware แนบข้อมูลระดับ request (เช่น user ปัจจุบัน) ไว้ที่ `Astro.locals` คุณ type มันด้วยการประกาศ interface `App.Locals` มักอยู่ใน `src/env.d.ts`:

```ts
// src/env.d.ts
declare namespace App {
  interface Locals {
    user: { id: string; name: string } | null;
  }
}
```

ตอนนี้ `Astro.locals.user` มี type ทุกที่ — ใน middleware, page และ endpoint

## type-check ด้วย astro check

editor type-check ให้ระหว่างเขียน แต่ไฟล์ `.astro` ไม่ถูก check โดย `tsc` คำสั่งที่ทำคือ **`astro check`** — มัน validate type ข้ามไฟล์ `.astro`, `.ts` และ `.tsx` ต่อมันเข้ากับ build เพื่อให้ type error หยุด deploy ที่พัง:

```json
{
  "scripts": {
    "build": "astro check && astro build"
  }
}
```

<Callout title="สรุปหนึ่งบรรทัด">
Astro มี **TypeScript ในตัว** (extend `astro/tsconfigs/strict`) type props ของ component ด้วย interface `Props` และ type ข้อมูล request บน `Astro.locals` ผ่าน interface `App.Locals` ฟีเจอร์เด่น: Astro **generate type จาก Zod schema ของ content** ดังนั้น `getCollection`/`getEntry` คืน entry ที่มี type ครบ พร้อม autocomplete และ error เมื่อเข้าถึง field ผิด รัน **`astro check`** (ควรเป็น `astro check && astro build`) เพื่อ type-check ไฟล์ `.astro` และหยุด build ที่พัง
</Callout>
